What is DocHub?

DocHub is a powerful documentation platform that helps teams create, organize, and share their technical documentation. It supports multiple formats including Markdown, PlantUML, YAML, and more, allowing you to create comprehensive documentation that is both easy to write and beautiful to read.

Easy to Use

Create beautiful documentation with simple markup languages and intuitive interfaces.

Architecture Diagrams

Generate diagrams from code using PlantUML, Mermaid, and other diagramming tools.

API Documentation

Document your APIs with OpenAPI, AsyncAPI, and other industry-standard formats.

Component-Based

Organize your documentation into reusable components for better maintainability.

Extensible

Extend functionality with plugins and custom templates to fit your needs.

Collaborative

Work together with your team on documentation using version control and review tools.

Getting Started

Follow these simple steps to start using DocHub for your documentation needs:

Start by creating a new Markdown or YAML file in your documentation directory. You can use any text editor to create these files.

# Welcome to My Documentation

This is a simple Markdown document that will be rendered by DocHub.

## Features

- Easy to write
- Looks great
- Supports code blocks

```python
def hello_world():
    print("Hello, DocHub!")
```

Create a dochub.yaml file to define the structure of your documentation:

title: My Project Documentation
description: Documentation for my awesome project
version: 1.0.0
structure:
  - title: Introduction
    path: introduction.md
  - title: Architecture
    children:
      - title: Overview
        path: arch/overview.md
      - title: Components
        path: arch/components.md

DocHub supports PlantUML for creating diagrams:

@startuml
package "Frontend" {
  [Web App]
  [Mobile App]
}

package "Backend" {
  [API Gateway]
  [User Service]
  [Content Service]
  [Database]
}

[Web App] --> [API Gateway]
[Mobile App] --> [API Gateway]
[API Gateway] --> [User Service]
[API Gateway] --> [Content Service]
[User Service] --> [Database]
[Content Service] --> [Database]
@enduml

Build your documentation site and deploy it:

# Build the documentation
npm run build

# Deploy to your server
npm run deploy

Or use GitHub Pages for easy hosting:

# Deploy to GitHub Pages
npm run deploy:github

Explore Components

DocHub organizes architecture and documentation into reusable components that can be referenced across your documentation.

Component Type Description Format
Aspects Cross-cutting concerns and perspectives YAML
Components Architectural building blocks YAML
Contexts Bounded contexts and domains YAML, PlantUML
Documents Documentation pages and sections Markdown, YAML
Rules Validation and architectural rules YAML
